# Graph-RAG Demo

A demo of [Microsoft GraphRAG](https://github.com/microsoft/graphrag) — knowledge-graph-enhanced retrieval-augmented generation powered by OpenAI.

GraphRAG extracts entities and relationships from your documents, builds a community-detected knowledge graph, and uses it to answer both specific and broad thematic questions — something flat vector-search RAG can't do.

## How it works

```
Documents → Entity extraction → Knowledge graph → Community detection
                                                        ↓
Query ──────────────────────────────────────→ Graph-aware retrieval → LLM answer
```

- **Local search** — answers specific questions about named entities using the graph neighborhood
- **Global search** — answers thematic/broad questions by reasoning over the entire community structure

## Quickstart

### 1. Install dependencies

```bash
python -m venv .venv
source .venv/bin/activate        # Windows: .venv\Scripts\activate
pip install -r requirements.txt
```

### 2. Set your OpenAI API key

```bash
cp .env.example .env
# Edit .env and replace the placeholder with your actual key
```

### 3. Initialize GraphRAG prompts

```bash
python -m graphrag.index --init --root .
```

This generates the `prompts/` directory with default extraction prompts.

### 4. Index your documents

Put `.txt` files in the `input/` directory (a sample `ai_ecosystem.txt` is included), then run:

```bash
python -m graphrag.index --root .
```

Indexing calls OpenAI to extract entities, relationships, and community summaries — it may take a few minutes.

### 5. Query

**Global search** (broad / thematic questions):
```bash
python query.py "What are the major relationships between AI companies?"
```

**Local search** (specific entity questions):
```bash
python query.py "What products has Microsoft built with OpenAI?" --method local
```

Or use the graphrag CLI directly:
```bash
python -m graphrag.query --root . --method global "Who are the key players in the LLM ecosystem?"
python -m graphrag.query --root . --method local  "Tell me about Anthropic's safety research"
```

## Project structure

```
graph-rag/
├── input/               # Source documents to index
│   └── ai_ecosystem.txt # Sample document (AI company relationships)
├── prompts/             # Auto-generated by --init (entity/community prompts)
├── output/              # Auto-generated index artifacts (gitignored)
├── cache/               # LLM call cache (gitignored)
├── settings.yml         # GraphRAG configuration
├── .env.example         # Environment variable template
├── requirements.txt     # Python dependencies
└── query.py             # Convenience query wrapper
```

## Configuration

Edit `settings.yml` to change:
- `llm.model` — swap in `gpt-4o-mini` for cheaper indexing
- `embeddings.llm.model` — change the embedding model
- `chunks.size` / `chunks.overlap` — tune chunking strategy
- `entity_extraction.entity_types` — add domain-specific entity types

## Adding your own documents

Drop `.txt` files into `input/` and re-run `python -m graphrag.index --root .`. The cache means only new content is re-processed.
